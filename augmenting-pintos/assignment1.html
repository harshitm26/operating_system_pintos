Deadline: 28th August, 2012 (IST)

A penalty of 20% per day would be imposed for late submission.

This assignment is to be implemented in three person teams.

Objective

Part 1

Read, understand and summarize the POSIX message queues. A good place to begin is mq_overview.

Part 2

You are required to implement a subset of the POSIX interface to the message queues in Pintos. The following functions (with the exact prototype as provided in the POSIX interface) must be implemented:


mqd_t mq_open(const char *name, int oflag);

int mq_send(mqd_t mqdes, const char *msg_ptr, size_t msg_len, unsigned msg_prio);

ssize_t mq_receive(mqd_t mqdes, char *msg_ptr, size_t msg_len, unsigned *msg_prio);

int mq_close(mqd_t mqdes);


Note that the above functions must be implemented as system calls in Pintos. The parameter name in mq_open() may be any string to serve as a common identity for a message queue (i.e. it need not be a file-system path as required by the POSIX calls).

Part 3

Use your (above) implementation of message queues in Pintos to solve the Producer-Consumer problem (discussed in class).
